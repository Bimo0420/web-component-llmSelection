# Local LLM Selector Creator

Этот скилл предназначен для автоматизации создания инструментов подбора локальных моделей. Он объединяет знания об архитектурах LLM и принципах построения удобных интерфейсов для выбора моделей под конкретные задачи.

## Основной процесс создания панели

1. **Определение входных параметров**: Сбор данных о сценарии использования.
2. **Логика фильтрации**: Выбор подходящих моделей на основе сценариев (Reasoning, Architecture, Context).
3. **Сравнение по бенчмаркам**: Ранжирование отфильтрованных моделей по ключевым метрикам.
4. **Генерация UI**: Создание интерфейса с карточками моделей и графиками сравнения.

## 1. Сценарии использования и фильтры

Панель должна поддерживать пресеты фильтрации для типичных задач.
**Логика фильтрации:**

### Сценарий A: Сложный кодинг и Математика
- **Цель**: Решение алгоритмических задач, генерация кода, научные вычисления.
- **Фильтр**: 
  - `Capabilities.Reasoning` >= **Advanced** (или **Excellent**)
  - `Capabilities.Coding` >= **Advanced** (или **Excellent**)
- **Ключевые бенчмарки для сортировки**: LiveCodeBench, AIME 2026, MATH 500.

### Сценарий B: Высоконагруженный чат-бот
- **Цель**: Быстрые ответы множеству пользователей, экономия ресурсов.
- **Фильтр**: 
  - `Architecture` == **MoE** (Mixture of Experts) для быстрого инференса.
- **Ключевые бенчмарки**: TTFT (Time To First Token), La Perf.

### Сценарий C: Анализ больших архивов (RAG)
- **Цель**: Поиск информации в десятках/сотнях документов.
- **Фильтр**: 
  - `Context Window` >= **128,000** (128k+).
- **Ключевые бенчмарки**: IFEval (следование инструкциям в длинном контексте), MMLU-Pro.

## 2. Сравнение и бенчмарки

После первичной фильтрации по Сценарию, модели должны сравниваться по набору метрик:
- **Общие**: Open LLM Leaderboard v2 (Hugging Face).
- **Инструкции**: IFEval.
- **Знания/Язык**: MMLU-Pro, Ru-MMLU, VLigaBench-ru.
- **Производительность**: TTFT, La Perf.

## 3. Источник данных (Model Database)

**Важно**: База данных моделей не должна быть хардкодом. Скилл должен предполагать загрузку данных из внешнего файла проекта (например, `data/models.json` или `src/data/models.ts`).
См. структуру данных в [Model Data Reference](references/model-data.md).

## 4. UI Компоненты

- **Фильтр сценариев**: Dropdown или карточки выбора ("Кодинг", "Чат", "Документы").
- **Таблица сравнения**: Столбцы с бенчмарками, подсвечивание лидеров.
- **Детали модели**: Показ поддерживаемых контекстов и особенностей.
