| **Тип и путь файла**                                         | **Область действия (Scope)** | **Приоритет** | **Основное назначение**                                                                                          |
| ------------------------------------------------------------ | ---------------------------- | ------------- | ---------------------------------------------------------------------------------------------------------------- |
| **`C:\...\.gemini\GEMINI.md`**                               | **Глобальная**               | Низкий        | Базовые настройки личности ИИ, язык общения и глобальные запреты для всех проектов.                              |
| **`\GEMINI.md`** (в корне проекта)                           | **Проектная**                | Средний       | Общие стандарты кодинга для конкретного репозитория (например, "используй архитектуру Clean Code").              |
| **`.agent/rules/ui.md`**                                     | **Проектная (локальная)**    | **Высокий**   | Узкоспециализированные правила. ИИ обращается к ним при работе с UI-компонентами. Перекрывает общие `GEMINI.md`. |
| **`.agent/workflows/workspace-1.md`**                        | **Проектная**                | N/A (Процесс) | Пошаговый план выполнения сложной задачи (например, "Миграция БД"). Запускается как сценарий.                    |
| **`C:\...\.gemini\antigravity\global_workflows\globe-1.md`** | **Глобальная**               | N/A (Процесс) | Общие сценарии, доступные в любом проекте (например, "Создать новый микросервис из шаблона").                    |
| **`.agent/skills`** (директория)                             | **Проектная / Глобальная**   | Динамический  | Набор инструментов. Агент "подгружает" их в память только тогда, когда задача совпадает с описанием навыка.      |

1. Rules (.md в /rules/ или GEMINI.md): Это "декларативное" управление. Вы описываете состояние, к которому ИИ должен стремиться (например, "Код должен быть всегда документирован"). ИИ держит это в голове постоянно.
2. Workflows (.md в /workflows/): Это "процедурное" управление. Описывает последовательность действий (Шаг 1, Шаг 2, Проверка). Вы используете это, когда нужно провести агента по сложному пути, где важен порядок.
3. Skills (.agent/skills/): Это "модульное" расширение. В отличие от правил в папке rules, навык может содержать свои собственные скрипты (.sh, .py) и примеры. Это делает агента "умнее" в конкретной области без замусоривания его основной памяти (контекста) лишними деталями, пока они не нужны.
